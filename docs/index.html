NXFit SDK

This SDK is intended to be used to connect to NXFit services.

Getting Started

The NXFit SDK is currently hosted on GitHub Packages. Follow these steps to add the SDK to your build:

Create GitHub Access Token

GitHub provides maven packages only through authenticated connections.

  • Login to GitHub (create an account if needed).

  • Go to Settings → Developer Settings → Personal access tokens → Tokens (Classic)

  • Click on Generate new token (Classic)

  • Give the new token a meaningful name and the only scope required is read:packages.

  • Since this token only requires read:packages set the expiration to whatever you feel comfortable with, however for this token scope No Expiration should be fine.

  • Once the required scope is checked, click Generate token.

  • Copy the generated access token and record it somewhere safe. You'll need it when updating the build.gradle files.

Add the NXFit SDK GitHub Packages repository to your gradle build files

  • Using the access token generated in the prior step, add the following maven repository to settings.gradle file:

      dependencyResolutionManagement {
        // ...
        repositories {
            // other repositories....
    
            // For nxfit-sdk from GitHub Packages
            maven {
                name = "GitHubPackages"
                url = uri("https://maven.pkg.github.com/NeoEx-Developer/NXFit-SDK-for-Android")
    
                credentials {
                    username = YOUR_GITHUB_USERNAME
                    password = YOUR_GITHUB_ACCESS_TOKEN
                }
            }
        }
    }

  • Add the following dependency to your app's build.gradle.kts file:

      dependencies {
        implementation("com.neoex:nxfit-sdk:9.1.0")
    }
    Content copied to clipboard

  • Sync the project with Gradle files

  • The NXFit interface is the main starting point. The NXFit class must be instantiated using the NXFit.build function. The NXFit.build function requires the following: A reference to the Application Context, an instance of ConfigurationProvider, the current user ID and an access token flow. Note that all these values are expected to remain valid for the entire lifecycle of the NXFit instance, with only the access token being refreshed as needed through the access token flow.

  • See documentation for usage.

Change Log

Version 9.1.0

  • Refactored logging and introduced configurable log levels. See ConfigurationProvider.

Version 9.1.3 - 9.0.7

  • Updates to nxfit_sdk-healthconnect

Version 9.0.2

  • HealthConnectState now supports serialization. This is needed by the associated Sync SDK for Flutter.

Version 9.0.1

  • Changed minSdk for nxfit-sdk to 26 from 24

  • Updated documentation and fixed minor issues.

Version 9.0.0:

  • The NXFit.build static method is no longer a suspend method.

  • Added support for creating sessions: added SampleClient, SessionSampleChunkClient and SyncClient.

  • Added create() and completeSession() methods to SessionClient.

  • Added a number of models to support creating sessions but also for daily health samples (ex: NewBoneMassSampleModel, NewSleepLevelSampleModel, etc).

Version 8.0.0

  • Removed SourcesManager from :nxfit-sdk-repositories and replaced it with SourceRepository in :nxfit-sdk. Also added a SourceClient and updated the SourceModel and related DTOs to match the latest from the server API.

  • IntegrationsManager is now IntegrationManager.

  • Fixed error handling issue when adding integrations.

Version 7.3.0

  • Created new nxfit-sdk-repositories AAR library to contain all the NXFit Repositories.

  • Renamed SourceManager to SourcesManager to be more consistent with IntegrationsManager.

Version 7.2.0

  • No longer need to pass the Context into NXFitRepositories.build. Updated documentation to various parts of the SDK.

Version 7.1.0

  • Refactored implementation of Repositories into an implementation class and converted the Repositories into interfaces.

Version 7.0.0

  • Refactored package from com.nxfit.sdk to com.neoex.nxfit

Version 6.0.0

  • Changes to AuthProvider, IntegrationsManager, SourceManager and associated models.

Version 5.0.0

  • Changes to Sources and Sessions

Version 4.4.0

  • Updated to work with changes to the API around new ActivityTypes. Also, the ActivityType is now a String instead of an Int.

Version 4.3.1

  • Fixed issue that prevented syncing Sources under certain conditions. This issue was introduced in version 4.3.0.

Version 4.3.0

  • Added IntegrationsManager.disconnectionEvents and associated IntegrationsManager.DisconnectionEvent to allow client apps to react when a user disconnects an integration. Note that this is only trigger from a user action that leads to disconnection.

Version 4.2.0

  • Renamed the GarminManager to IntegrationsManager along with renaming associated classes and functions as appropriate.

  • Added IntegrationManager.connectionEvents which is used to emit integration connection events. This is triggered from IntegrationManager.handleGarminConsentResponse().

Version 4.1.0

  • Restored Restored ProfileClient.deleteImage() which was accidently removed in prior version.

Version 4.0.0

  • The PersonalSessionDetailModel and PublicSessionDetailModel models have been refactored to contain some metric data as sub-objects that could be null if the metric data is not present.

Version 3.5.0

  • Note, this version contained breaking changes and should have been bumped to version 4.0.0

  • A number of changes were made to many clients and models.

  • The ProfileModel has been renamed to PersonalProfileModel and a PublicProfileModel has been added.

  • A new BadgeModel has been added.

  • BadgesModel has been renamed to UserBadgesModel

  • SessionSummaryModel has been renamed to SessionMetricsModel

  • The ProfileClient.get() overrides have been replaced with a single getPublicProfile() method.

  • a new userId: Int parameter has been added to the PublicSessionClient/PublicSessionSampleClient calls that get a session or metrics/samples for a session.

  • SessionClient.listHeartRateZones() has been removed and it's functionality can be replaced with SessionClient.listHeartRateZoneSummaries().

  • SessionSummaryClient.getMovement() has been removed along with the MovementSummaryModel as the contents of that model have been moved to SessionSummaryModel.

Version 3.4.0

  • Added support for paged data.

  • Renamed PublicSessionModel to PublicSessionDetailModel

  • Renamed SessionModel to PersonalSessionDetailModel

  • Updated SessionClient and PublicSessionClient to support retrieving lists of sessions in a format that supports paged data.

  • Added PaginationQuery that allows to query for paged data.

Version 3.3.1

  • ProfileClient.updateImage now returns the Uri of the new profile image.

Version 3.3.0

  • Profile image is now restricted in size and compressed with JPEG before being sent to the server.

Version 3.2.0

  • Added SourceManager.sync() to allow client applications to trigger a sync when needed. This is mostly to allow user initiated refresh ex: Pull to Refresh. This does not invalidate caches.

  • SourceManager.get() now returns SourceModel instead of SourceModel?.

  • Added SourceManager.getOrNull()

Version 3.1.1

  • Fixed issue where the GarminManager would attempt to retrieve Garmin state before the user was authenticated.

Version 3.1.0

  • Added Cadence and Power fields to the SessionModel.

  • Added Cadence and Power sample support.

  • Removed unsupported parameters from Session Sample queries.

  • SessionModel.SourceId has been converted from String to Int.

Version 3.0.0

  • Identifiers in various models have been converted from UUID to Int.

Version 2.0.0

  • Updated models to reflect changes to the API. This is mostly the sample models, but also the SessionModel and SessionsModel. Also, Sample models now have an activeTime property.

  • Added helper methods to some of the clients to make it easier to fetch data that controls cache with both If-Modified-Since and eTag headers.

  • Fixed bug in Badges client that would not properly use the eTag

Version 1.3.0

  • BadgeModel now represents the progress as a value between 0.0f to 1.0f

  • PublicSessionClient has new methods: listHeartRateZones() and listHeartRateZoneSummaries()

  • Added PublicSessionSampleClient

Version 1.2.0

  • PublicSessionProtocol.list has new parameter: studio.

  • PublicSessionProtocol.get no longer accepts both eTag and ifModifiedSince parameters.

  • Added Studio enum to distinguish public sessions between the user's studio and all studios

  • Added isUserSession property to both PublicSessionModel and PublicSessionsModel.PublicSession
