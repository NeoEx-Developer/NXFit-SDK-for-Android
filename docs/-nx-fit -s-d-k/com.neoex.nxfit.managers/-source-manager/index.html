SourceManager

interface SourceManager

Maintains the state of health and session sources and is designed to be used as a singleton.

When instantiated, a sync process is triggered to ensure the cached list of sources matches the server. If there is no authenticated user at the time then the sync process is deferred until the user authenticates. The sync process will run again when ever the user re-authenticates, when the user connects with Garmin Connect, when a user disconnects from Garmin Connect, or a call to update is made.

A User may have one or more sources of health/session data, not only for a connection such as HealthKit but also within that connection. E.g. HealthKit may have multiple sources for the User's health data.

Sources can be prioritized so that when overlapping workout session data exists from multiple sources, the session data from the source with highest priority is used. A session is considered to overlap based on start and end datetime.
