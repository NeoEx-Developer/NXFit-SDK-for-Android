disconnect

abstract suspend fun disconnect()

Disconnects the user from this integration. After a successful disconnect, isConnected will be false. This method is called by the IntegrationsManager when IntegrationsManager.disconnect is invoked for this specific integration.

Disconnecting should clean up any local state associated with the connection. However, it's not necessary to revoke permissions that the user may have granted. For example, if a user disconnects from Health Connect, the app may retain any permissions it was granted, allowing for a smoother reconnection process in the future.

Throws

IntegrationDisconnectException

if the disconnection fails.
